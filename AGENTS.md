- [2026-04-19] nitro runtime verification: `app/server.ts` is picked up by Nitro production builds, but the plain `npm run dev` Vite server does not exercise those server-entry routes the same way. When changing `/.well-known/*`, custom headers, or `Accept: text/markdown` behavior, verify with `npm run build` plus `npm start` (or the deployed runtime), not only with the dev server.
- [2026-04-19] nitro query routing: In Nitro handlers here, `event.path` includes the query string, so exact route checks like `event.path === "/api/services"` fail for `?category=` requests. When matching API routes that accept query params, parse `new URL(event.req.url, siteCanonical)` and compare `url.pathname` instead.
